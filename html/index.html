📖 WebProject Studio 사용 가이드
🏠 이 도구가 뭐예요?
WebProject Studio는 웹사이트를 만들기 전에 "어떤 화면을 만들지" 미리 계획하고, AI한테 부탁하는 문서를 자동으로 만들어주는 도구예요.

쉽게 말하면, 레고 설명서 같은 거예요! 🧩 레고를 조립하기 전에 설명서를 보듯이, 웹사이트를 만들기 전에 이 도구로 설계도를 만드는 거예요.
✅ 설치 없이 그냥 파일을 열면 바로 써요!
✅ 인터넷 없어도 돼요 (오프라인 OK)
✅ 내가 입력한 내용은 자동으로 저장돼요
✅ Chrome, Edge, Safari 아무 브라우저나 OK
바탕화면에 이 파일을 저장해두고 매번 브라우저로 열어서 쓰면 편해요!
📱 화면에 뭐가 있어요?
화면 위쪽에 탭이 4개 있어요. 탭마다 다른 역할을 해요.
1
📊 대시보드 — 프로젝트가 얼마나 됐는지 진행 현황을 한눈에 볼 수 있어요. "이 페이지는 완료, 저 페이지는 아직" 같은 걸 정리해줘요.
2
✨ AI 프롬프트 — 가장 많이 쓰는 탭! 정보를 입력하면 AI(Claude, Figma)한테 보낼 설계 문서를 자동으로 만들어줘요.
3
📄 페이지 생성 — 각 페이지의 설명 문서(마크다운 파일)를 뚝딱 만들어줘요.
4
🗂 MD Vault — 만든 문서들을 폴더에 정리하고 편집할 수 있는 메모장이에요.
✨ 처음 쓸 때 이렇게 해보세요!
가장 많이 쓰는 기능인 AI 프롬프트 탭 기준으로 설명할게요.
1
왼쪽 사이드바에서 "+ 새 프로젝트" 버튼을 눌러요. 프로젝트 이름을 입력하고 Enter!
2
① 기본 설정 탭에서 프로젝트 이름, 폰트, 화면 크기 같은 기본 정보를 입력해요. 처음엔 그냥 기본값 두고 넘어가도 돼요!
3
② 페이지 구성에서 만들 페이지들을 추가해요. 예: Home, About, Contact 이런 식으로요. ID는 01, 02, 03 순서대로!
4
③ 섹션 구성에서 각 페이지 안에 뭐가 들어갈지 골라요. 예: Hero(메인 배너), 카드 그리드, 문의 폼 등등!
5
✨ 프롬프트 생성 버튼을 누르면 AI한테 보낼 설계서가 자동으로 만들어져요!
6
만들어진 글을 복사해서 Claude 채팅창에 붙여넣기 하면 끝! AI가 알아서 와이어프레임을 만들어줘요. 🎉

이게 전부예요! 어렵지 않죠? 더 자세한 내용이 궁금하면 위에 "🔵 디테일 버전" 버튼을 눌러보세요.
1
WebProject Studio란?
웹 프로젝트 기획·설계의 전 과정을 HTML 파일 하나에서 처리할 수 있는 올인원 도구입니다. 설치·서버·인터넷 연결이 전혀 필요 없습니다.
✔ 설치 불필요 — 파일을 브라우저로 열면 즉시 실행됩니다. Chrome / Edge / Safari 모두 지원합니다.
✔ 오프라인 동작 — 별도 서버·앱·인터넷 연결 없이 로컬에서 완전히 작동합니다.
✔ 자동 저장 — 모든 데이터는 브라우저 localStorage에 저장되어 새로고침·재시작 후에도 유지됩니다.
✔ 데이터 이동 — 다른 PC에서 사용하려면 슬롯 내보내기(JSON) 또는 MD 파일로 내보낸 뒤 가져오기 하세요.
💡 바탕화면이나 팀 공유 폴더에 파일을 두고 브라우저 북마크에 등록해두면 편합니다. 브라우저 캐시를 전체 삭제하면 localStorage 데이터도 사라질 수 있으니 중요한 데이터는 MD 파일로 주기적으로 백업하세요.
2
상단 탭 4개 — 각 탭의 역할
📊
대시보드
sitemap.md의 페이지 상태표를 붙여넣으면 기획·디자인·개발 역할별 완료/진행중/미착수를 자동 계산하고, 전체 진행률 게이지와 페이지 테이블을 시각화합니다. 진행 보고서(MD) 다운로드도 가능합니다.
✨
AI 프롬프트
4단계(기본 설정 → 페이지 구성 → 섹션 구성 → 푸터 설정) 입력 후 버튼 하나로 Claude 업로드용, Figma AI용, MD 파일용 프롬프트를 동시에 생성합니다. 슬롯 시스템으로 설정을 폴더별 보관·불러오기 가능합니다.
📄
페이지 생성
페이지 ID·이름·섹션 목록 등 폼을 채우고 "📄 파일 생성"을 누르면 구조화된 마크다운 페이지 문서가 즉시 생성됩니다. Works·Story 프리셋으로 바로 결과물을 확인할 수 있습니다.
🗂
MD Vault
브라우저 안에서 마크다운 문서를 작성·편집·미리보기·폴더 관리할 수 있는 내장 에디터입니다. AI 프롬프트 결과를 "🗂 MD Vault에 저장"하면 자동으로 이 탭으로 이동하고 파일이 열립니다. 외부 MD 파일 가져오기·내보내기도 지원합니다.
3
좌측 사이드바 — 프로젝트 히스토리
AI 프롬프트 탭에서 작업한 내용이 자동으로 사이드바에 기록됩니다. 최대 30개 프로젝트를 보관할 수 있으며, 30개를 초과하면 가장 오래된 항목이 자동 삭제됩니다.
새 프로젝트 버튼새 빈 항목을 생성하고 이름 편집 모드를 자동으로 시작합니다
항목 클릭해당 프로젝트의 모든 설정(기본·페이지·섹션·푸터)을 불러옵니다
항목 더블클릭프로젝트 이름 직접 수정 (Enter로 확정, Esc로 취소)
× 버튼해당 프로젝트 삭제 (복구 불가)
이름 옆 ● 점🟡 저장 중 → 🟢 저장 완료 (자동저장 상태 표시)
히스토리는 AI 프롬프트 탭의 설정값만 저장합니다. 대시보드 파싱 결과·Vault 파일은 별도로 관리됩니다.
4
실시간 자동저장
AI 프롬프트 탭에서 값을 변경하면 0.8초 후 자동으로 저장됩니다. 페이지 추가·삭제, 섹션 추가·삭제, 드래그 순서 변경 시에도 즉시 반영됩니다.
⚠ 활성 프로젝트(사이드바에서 선택된 항목)가 없으면 자동저장이 작동하지 않습니다.
먼저 "새 프로젝트" 버튼을 누르거나, 히스토리에서 항목을 클릭해 불러온 뒤 작업을 시작하세요.
사이드바에 선택된 항목이 있으면 이름 옆 ● 점이 녹색으로 표시됩니다.
5

전체 삭제슬롯 매니저 우측 상단 "🗑 전체 삭제"
1
먼저 T51 샘플로 전체 구조 파악
"📌 T51 샘플 불러오기" 버튼을 누르면 실제 T51 프로젝트 데이터(기획·디자인·개발 역할별 상태 포함)가 자동으로 채워지고 대시보드가 즉시 생성됩니다.
전체 진행률 게이지 ✅ 완료 카운트 (역할별) 🚧 진행중 카운트 (역할별) ⬜ 미착수 카운트 (역할별) 페이지별 상태 테이블
샘플로 전체 구조를 먼저 파악한 뒤, 실제 sitemap 데이터로 교체해서 사용하세요. "🗑 초기화" 버튼으로 언제든 리셋할 수 있습니다.
2
실제 sitemap 데이터 붙여넣기
관리 중인 sitemap.md에서 페이지 상태표 부분을 전체 복사 → 텍스트박스에 붙여넣기 → "🔍 파싱해서 대시보드 생성" 클릭
지원하는 마크다운 표 형식 — 2가지
① 역할별 분리형 (권장)
| ID | 페이지명 | 섹션 수 | 기획 | 디자인 | 개발 | 우선순위 | 비고 |
| 01 | Home | 9 | ✅ done | 🚧 wip | ⬜ todo | P1 | |

② 통합 상태형
| ID | 페이지명 | 섹션 수 | 상태 | 우선순위 | 비고 |
| 01 | Home | 9 | ✅ done | P1 | |

표의 구분선 행(|---|)이 있으면 자동으로 인식합니다. 불필요한 앞뒤 텍스트가 있어도 표 부분만 추출합니다.
3
사용 가능한 상태값
✅ done 완료
🚧 wip 진행중
⬜ todo 미착수
🔴 hold 보류
이모지 없이 done / wip / todo / hold 텍스트만 입력해도 인식됩니다. 대소문자 구분 없이 파싱됩니다.
